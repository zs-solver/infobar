# 边缘手柄三区域交互 + 删除操作拆分 — 需求文档

## 背景

当前 InfoBar 的边缘手柄（EdgeHandle）是 5px 宽的窄条，拖拽时按比例缩放所有格子。用户希望将多种操作集成到边缘手柄上，取代之前的全局「布局模式」切换方案。同时将右键菜单的「删除此格子」拆分为两种行为。

## 需求一：删除操作拆分

将右键菜单中的「删除此格子」拆为两个并列选项：

| 菜单项 | 行为 |
|--------|------|
| **删除此格子（窗口缩小）** | 删掉格子后，其他格子宽度不变，窗口总宽度减小 |
| **删除此格子（宽度不变）** | 删掉格子后，窗口总宽度不变，剩余格子重新分配空间 |

- 只剩 1 个格子时，两个选项均禁用

## 需求二：边缘手柄三区域交互

将左右两侧的 EdgeHandle 从单一拖拽行为改为三区域设计，从上到下分布三种功能：

### 2.1 区域划分（纵向）

```
┌─────┐ ← 上区域：插入新格子（点击）
│  +  │
├─────┤
│     │
│  ⇔  │ ← 中区域：移动总宽度（拖拽）
│     │
├─────┤
│  ↔  │ ← 下区域：拖拽增删格子
└─────┘
```

- **上区域（EDGE_ZONE_HEIGHT 高）**：点击在该侧边缘插入一个新格子
- **中区域（剩余高度）**：拖拽仅移动窗口边缘总宽度，所有格子始终保持平均宽度，不单独调整边缘格子
- **下区域（EDGE_ZONE_HEIGHT 高）**：拖拽可裂变新增空格子（拉伸时）或缩减删除空格子（收缩时）

### 2.2 悬停展开

| 状态 | 宽度 | 高度 | 说明 |
|------|------|------|------|
| 默认 | 5px | 文本高度 | 仅显示中区域的窄条，与当前外观一致 |
| 鼠标悬停 | 24px | 文本高度 + 2 × 44px | 上下区域从文本区域外"长出来"，窗口仅向下扩展，不跳动 |
| 鼠标离开 | 5px | 文本高度 | 缩回原始宽度和高度 |

**关键约束：中区域的高度始终不变**。悬停时上下区域向外长出，中区域本身不被压缩。窗口只向下扩展，不移动 Y 坐标，避免视觉跳动。

### 2.3 各区域详细行为

#### 上区域 — 插入新格子（点击）

- 左侧手柄：在最左侧插入新格子
- 右侧手柄：在最右侧插入新格子
- 插入后窗口总宽度增大（已有格子宽度不变），新格子宽度 = 当前各格子平均宽度
- 鼠标样式：`PointingHandCursor`

#### 中区域 — 移动总宽度（拖拽）

- 拖拽时改变窗口总宽度，所有格子始终保持平均等宽
- 不单独调整边缘格子宽度（区别于旧版行为）
- 鼠标样式：`SizeHorCursor`

#### 下区域 — 拖拽增删格子（拖拽）

- **拉伸方向**（向外拖）：边缘格子变宽，当边缘格子宽度 ≥ 标准宽度 × 2 时裂变出新空格子
- **收缩方向**（向内拖）：边缘空格子缩窄，宽度 ≤ 阈值时自动删除；有内容的格子不删除，仅缩到最小宽度
- 鼠标样式：`SizeHorCursor`

### 2.4 右键菜单调整

由于「插入新格子」操作移到了边缘手柄上，右键菜单中的「在左侧插入新格子」和「在右侧插入新格子」**保留不变**（两处入口并存，不冲突）。

## 涉及文件

| 文件 | 改动 |
|------|------|
| `widgets.py` | EdgeHandle 重写为三区域设计（悬停展开、区域判定、三种交互逻辑）；新增常量；EditableField 右键菜单拆分删除选项 |
| `info_bar.py` | 新增 `_create_field` 工厂方法、`_get_non_splitter_width`、`insert_at_edge`、`delete_field_shrink`、`delete_field_keep_width`、`notify_handle_expanded/collapsed`；重构 `insert_field_at`、`sync_heights`、`save_config` |
| `storage.py` | 无改动 |

---

## 实现总结

### widgets.py 改动

**新增常量：**
- `EDGE_HANDLE_DEFAULT_WIDTH = 5` — 默认宽度
- `EDGE_HANDLE_EXPANDED_WIDTH = 24` — 悬停展开宽度
- `EDGE_ZONE_HEIGHT = 44` — 上下区域高度
- `SPLIT_FACTOR = 2.0` — 裂变阈值倍数
- `MIN_FIELD_WIDTH = 30` — 格子最小宽度

**EdgeHandle 三区域重写：**
- `set_base_height(h)` — 由 InfoBar 设置文本基准高度，手柄自行管理展开/收回高度
- `_zone_rects()` — 返回插入区/中区/裂变区三个 QRect，展开时从文本区下方长出
- `_hit_zone(pos)` — 判断鼠标位于哪个区域
- `enterEvent/leaveEvent/_do_collapse` — 悬停展开（宽5→24，高增加88px向下），带80ms防闪烁延迟
- `paintEvent` — 展开时三区域分色绘制，插入区画"+"号，裂变区画"↔"双箭头
- `_handle_middle_drag` — 中区域拖拽：改变窗口总宽度，所有格子保持平均等宽
- `_handle_bottom_drag` — 下区域拖拽：拉伸时边缘格子变宽并裂变，收缩时空格子自动删除

**EditableField 右键菜单：**
- 「删除此格子」拆分为「删除此格子（窗口缩小）」和「删除此格子（宽度不变）」
- 只剩 1 个格子时两个选项均禁用

### info_bar.py 改动

**新增方法：**
- `_create_field(text, font)` — 工厂方法，统一创建 EditableField
- `_get_non_splitter_width()` — 获取非 splitter 部分宽度
- `insert_at_edge(side, text, adjust_window)` — 在最左/最右插入新格子
- `delete_field_shrink(target_field)` — 删除格子并缩小窗口
- `delete_field_keep_width(target_field)` — 删除格子但保持窗口宽度
- `notify_handle_expanded()` — 手柄展开时窗口仅向下扩展
- `notify_handle_collapsed()` — 手柄收回时恢复窗口高度

**重构：**
- `sync_heights` — 改用 `set_base_height()` 通知手柄，而非直接 `setFixedHeight`
- `insert_field_at` — 使用工厂方法，插入时保持已有格子宽度不变
- `save_config` — 保存时始终使用 `_base_height` 作为窗口高度，避免展开状态下保存错误值
