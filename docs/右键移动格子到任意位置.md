# 右键移动格子到任意位置 — 需求文档

## 背景

当前 InfoBar 的右键菜单「移动」子菜单提供四个固定选项：到最左、向左一位、向右一位、到最右。当格子数量较多（≥5）时，若要把一个格子移动到中间某个位置，需要多次点击「向左/右一位」，操作繁琐。

## 需求

一步到位地将任意格子移动到任意目标位置，并提供直观的视觉参考。

### 核心设计

1. **动态位置列表**：「移动」子菜单根据当前格子数量动态生成所有可选位置，每个位置用相邻格子的圆圈序号标识
2. **序号覆盖层**：展开「移动」子菜单时，在每个格子上显示金色圆圈数字（①②③…）作为视觉参考，关闭菜单后自动清除
3. **兼容标注**：「到最左」「到最右」「向左一位」「向右一位」作为括号标注附在对应位置项后面

### 菜单结构示例

以 7 个格子 `[①A] [②B] [③C] [④D] [⑤E] [⑥F] [⑦G]`，右键点击 ④D 为例：

```
移动 ▸
  ① 之前（到最左）
  ① 和 ② 之间
  ② 和 ③ 之间（向左一位）
  ── 当前位置 ④ ──          ← 灰色不可点击，作为视觉锚点
  ⑤ 和 ⑥ 之间（向右一位）
  ⑥ 和 ⑦ 之间
  ⑦ 之后（到最右）
```

标注规则：
- 第一项（k=0）追加「到最左」
- 最末项（k=n-1）追加「到最右」
- 当前位置的前一项追加「向左一位」
- 当前位置的后一项追加「向右一位」
- 标注可叠加，如格子只有 3 个时可能出现「① 之前（到最左 · 向左一位）」

### 序号覆盖层

| 属性 | 值 |
|------|------|
| 显示时机 | 移动子菜单 `aboutToShow` 时创建，`aboutToHide` 时销毁 |
| 外观 | 圆形徽标，黑色半透明背景 + 金色圆圈数字 |
| 字体 | 与格子字体一致，尺寸 = 字体高度 + 10px |
| 位置 | 居中显示在各格子上 |
| 字符 | ① - ⑳（Unicode U+2460 - U+2473），超过 20 降级为 `(21)` 格式 |

### 边界情况

| 场景 | 行为 |
|------|------|
| 只有 1 个格子 | 移动子菜单整体禁用 |
| 只有 2 个格子 | 仅显示 1 个可操作项 + 当前位置，标注叠加（如「到最右 · 向右一位」） |
| 当前格子已在最左/最右 | 对应的「到最左/到最右」不出现（因为与当前位置重合，已被 disabled 项替代） |

## 涉及文件

| 文件 | 改动 |
|------|------|
| `info_bar.py` | 新增 `_position_overlays` 列表、`move_field_to`、`show_position_overlays`、`hide_position_overlays` 方法 |
| `widgets.py` | 右键菜单的移动子菜单从静态四选项改为动态位置列表，连接 `aboutToShow/aboutToHide` 信号 |

---

## 实现总结

### info_bar.py 改动

**新增属性：**
- `_position_overlays = []` — 存储覆盖层 QLabel 引用，用于统一清理

**新增方法：**
- `move_field_to(target_field, target_index)` — 移动格子到指定索引位置，按字段身份（而非位置顺序）保留各格子原有宽度，修复了旧 `move_field` + `rebuild_splitter` 中宽度恢复顺序错误的问题
- `show_position_overlays()` — 在每个字段上创建圆形徽标 QLabel（parent=field），字体与字段一致，`QFontMetrics` 动态计算尺寸
- `hide_position_overlays()` — 销毁所有覆盖层并清空列表，支持重复调用

### widgets.py 改动

**移动子菜单重构：**
- 用 `_circled(num)` 辅助函数生成圆圈数字
- 遍历 `range(n)` 动态构建菜单项，根据目标索引 `k` 与当前索引 `c` 的关系生成位置描述
- 位置描述逻辑：
  - `k < c` 时引用原始索引 `k` 和 `k+1` 的标号
  - `k > c` 时引用原始索引 `k+1` 和 `k+2` 的标号（因为弹出当前元素后索引偏移）
  - `k == 0`：特殊描述「① 之前」
  - `k == n-1`：特殊描述「⑦ 之后」
- 通过 `lambda checked, t=k` 正确捕获循环变量
- 连接 `move_menu.aboutToShow` → `show_position_overlays`
- 连接 `move_menu.aboutToHide` + `menu.aboutToHide` → `hide_position_overlays`（双重保险）
